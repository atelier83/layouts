# Contributing to @atelier83/layouts

Thank you for taking the time to contribute. This document covers everything you need to get the project running locally, make changes, and ship a release.

## Prerequisites

- [Node.js](https://nodejs.org/) ≥ 20
- [pnpm](https://pnpm.io/) ≥ 10 (`npm install -g pnpm`)

## Setup

```bash
git clone https://github.com/atelier83/layouts.git
cd layouts
pnpm install
```

## Development

```bash
pnpm dev          # start Vite dev server with the playground
pnpm test         # run tests once (vitest)
pnpm test:watch   # run tests in watch mode
pnpm check-types  # TypeScript type check (no emit)
pnpm build        # compile to dist/ and copy theme.css
```

The playground lives in `playground/` and hot-reloads automatically against `src/`.

## Project structure

```
src/
  core/       # headless engine — no DOM, no framework
  react/      # optional React bindings (LayoutProvider, usePanel)
  theme.css   # default dark theme (--a83-* tokens + skin rules)
test/         # vitest test suite
playground/   # local dev demo
dist/         # compiled output (git-ignored, generated by build)
.changeset/   # pending release notes (managed by changesets)
```

## Making changes

1. Create a branch: `git checkout -b feat/my-change`
2. Make your changes and add tests where relevant.
3. Run the full suite: `pnpm test && pnpm check-types`
4. Create a changeset describing what you changed:

   ```bash
   pnpm changeset
   ```

   You will be prompted to choose a bump type:
   - **patch** — bug fixes, style tweaks, documentation
   - **minor** — new features, non-breaking additions
   - **major** — breaking changes

   A markdown file is written to `.changeset/`. Commit it alongside your code.

5. Open a pull request against `main`.

## Releasing a new version

> This is done by a maintainer once changes are merged.

**1. Bump the version and generate the changelog:**

```bash
pnpm version
```

This reads all pending changeset files, bumps `package.json`, writes/updates `CHANGELOG.md`, and deletes the consumed changesets. Commit the result:

```bash
git add . && git commit -m "chore: release vX.Y.Z"
git push
```

**2. Publish to npm:**

```bash
pnpm release --otp=<your-2fa-code>
```

This runs `pnpm test && pnpm build && pnpm publish` in sequence.

**3. Tag and create a GitHub release:**

```bash
git tag vX.Y.Z && git push origin vX.Y.Z
gh release create vX.Y.Z --title "vX.Y.Z" --notes "$(sed -n '/^## X.Y.Z/,/^## /p' CHANGELOG.md | head -n -1)"
```

## Code style

- TypeScript strict mode is enabled — no `any`, no implicit returns.
- Private class members use `#` (native private fields), not `_` prefixes.
- The headless core (`src/core/`) must remain DOM-free and dependency-free.
- UI features go in `src/ui/` (or equivalent); framework bindings go in `src/react/`.
- CSS uses `--a83-*` tokens only — no hardcoded colour values in component rules.

## Reporting issues

Please open an issue at [github.com/atelier83/layouts/issues](https://github.com/atelier83/layouts/issues) and include:

- A minimal reproduction (JSFiddle, CodeSandbox, or a failing test case)
- Your browser/Node version
- What you expected vs. what happened
